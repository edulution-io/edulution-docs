## Verwendung

### Mit Markdown-Datei

```jsx
import { ChangelogFromMarkdown } from '@site/src/components/ChangelogFromMarkdown';
import changelogContent from '!!raw-loader!../changelogs/CHANGELOG.md';

<ChangelogFromMarkdown markdown={changelogContent} />;
```

### Mit direkten Daten

```jsx
import { Changelog } from '@site/src/components/Changelog';

<Changelog
  entries={[
    {
      version: '1.0.0',
      date: '2025-01-15',
      title: 'Erste Version',
      description: 'Beschreibung der ersten Version',
      tag: 'edulution-ui',
      improvements: ['Feature 1', 'Feature 2'],
      links: [{ text: 'Mehr Info', url: '/docs' }],
    },
  ]}
/>;
```

## Markdown-Format

Die Markdown-Datei sollte folgendes Format haben (nur EIN Tag pro Release):

```markdown
## v1.2.0 | 2024-03-15 | edulution-ui

### Feature Title

Description of the feature or changes.

#### Verbesserungen

- Improvement 1
- Improvement 2

[Link Text](https://example.com)

---

## v1.1.0 | 2024-02-01 | edulution-fileproxy

...
```
